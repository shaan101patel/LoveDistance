# LoveDistance Core User Flows

## 1. Pairing flow
1. User signs up.
2. User creates or enters a couple invite link.
3. Partner accepts invite.
4. Couple profile is created.
5. App lands on shared home screen.

## 2. Morning check-in flow
1. User opens app after waking up.
2. User logs wake-up / sleep check-in.
3. That day’s prompt unlocks.
4. User can answer immediately or later.

## 3. Daily prompt flow
1. Home shows question of the day.
2. User answers privately.
3. User sees own answer and blurred partner state.
4. Once both answer, thread unlocks.
5. Couple reacts, comments, or follows up.

## 4. Photo presence flow
1. User taps quick share.
2. User takes or uploads a photo.
3. Optional caption, mood, or location is added.
4. Photo appears in partner feed and latest-home surface.
5. Partner reacts or replies.

## 5. Habit flow
1. User opens calendar or today card.
2. User completes mine / yours / ours habit.
3. Shared completion logic updates.
4. Calendar state, streaks, and goals refresh.

## 6. Memory timeline flow
1. User opens timeline.
2. User filters by prompts, photos, gratitude, milestones, or favorites.
3. User opens an entry.
4. User can share link, favorite it, or revisit context.

## 7. Reunion countdown flow
1. Couple sets next visit date.
2. Home/widget can show countdown.
3. Milestone notifications trigger before reunion.

## 8. Notification flow
1. System decides a notification-worthy event.
2. It checks user preferences and local time zone.
3. Notification is sent or suppressed.
4. Tap opens directly to the correct screen.
